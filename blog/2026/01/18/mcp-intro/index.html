MCP 是什么：Model Context Protocol 与「工具化」的 AI

约 1169 字大约 4 分钟

MCPAI

2026-01-18

MCP（Model Context Protocol） 可以理解为：让 AI 客户端（例如 Cursor、Claude Desktop 等）用一套统一方式去连接外部能力——文件系统、数据库、Issue 系统、内部 API 等——的开放协议。目标不是再发明一种「聊天格式」，而是把 上下文（Context） 和 工具（Tools） 标准化，减少每个产品各自对接一遍的碎片成本。

这篇把概念拆细一点：角色分工、协议里到底有什么、传输方式、和插件的差别，最后落到你怎么用、写 Server 时要盯什么。

一、解决什么问题

没有 MCP 时，每个应用往往要自己实现：

  • 怎么安全地读仓库、列目录；
  • 怎么调 Slack / Jira / 内部网关；
  • 怎么把「工具返回」塞进模型上下文。

MCP 把这些抽象成 Server：对外声明 Resources（资源）、Tools（工具）、Prompts（可复用提示模板） 等，客户端用同一套流程 发现（list）与调用（call）。

直观收益：

  • 同一份 MCP Server 可在多个支持 MCP 的客户端里复用；
  • 团队可把内部系统封装成 Server，在 授权范围 内给助手用；
  • 模型侧往往以 function calling / tool use 呈现，但 Wire 格式与生命周期 由 MCP 规范，而不是每家自定义一坨 JSON。

二、架构里的三个角色

  • Host：运行 AI 的应用（编辑器、桌面客户端）。负责拉起 UI、模型、以及 MCP Client。
  • Client：Host 内与 MCP Server 通信的组件：握手、能力协商、转发请求。
  • Server：真正实现「读 Git」「查 Postgres」「拉 Linear」的一方；通过协议声明自己提供什么。

你日常「配 MCP」时，多数是在告诉 Host：用什么命令/URL 启动哪个 Server、传哪些环境变量。

三、协议里常见的「能力类型」（概念层）

具体字段以你所用的 SDK / 文档版本为准，但心智模型可以固定为：

类型典型用途模型侧体验
Resources暴露可读内容（文件、URL、数据库片段）「先读到上下文再回答」
Tools声明可调用的动作（查询、创建、执行脚本）类似函数调用，带参数与返回值
Prompts预置模板（如「按我司规范写 PR 描述」）减少重复 system 话术

关键点：Tools 往往伴随 副作用（写库、发消息），所以 权限、审计、幂等 要在 Server 实现里想清楚，不能指望模型「自觉」。

四、传输：stdio 与 HTTP/SSE

  • stdio：父进程拉起子进程，标准输入输出当信道。适合本地小工具、CLI 包装；部署简单，适合个人与内网脚本。
  • HTTP + SSE：远程服务、多客户端共享、需要独立扩缩容时更常见。

选型粗规则：本地-only、敏感数据不出机 → 优先考虑 stdio + 严格路径白名单；多团队共享、集中鉴权 → HTTP + 网关 + 审计。

五、怎么「用」起来（使用者 checklist）

  1. 在客户端里 添加 MCP Server（配置里通常有：启动命令、参数、env）。
  2. 本机先 手动跑一遍 同样命令，确认无缺失依赖、路径可读。
  3. 对话里 显式要求走某类能力，并核对 工具返回原文（不要只看模型总结）。
  4. 对敏感操作：默认只读 Server，写操作单独审批或二次确认（取决于你们网关能力）。

六、和「插件 / 直连 API」的差别

  • 传统 IDE 插件：常和某个编辑器 API 强绑定，换工具要重写。
  • 直连业务 HTTP：快，但每个产品都要写一遍「怎么暴露给模型、怎么鉴权、怎么截断上下文」。
  • MCP：在 互操作 上多了一层标准；Server 可独立演进，Host 只实现一次 Client。

七、写 Server 时多想的四层

  1. 最小权限：读哪些路径、调哪些 API、能否写。
  2. 错误语义：超时、4xx/5xx、部分成功——要让模型和人类都能排障。
  3. 输出截断：大结果要分页或摘要，避免撑爆上下文。
  4. 审计：谁在什么时间调用了什么工具（尤其是写操作）。

八、一句话收束

若你只做业务开发、不自己写 Server，记住一句就够：MCP 让 AI 在可控前提下多了一只「连到你系统的手」——手伸多远，由 Server 与权限决定；手伸出去之后的结果，仍以工具返回与审计日志为准。

更偏「配置示例、安全清单、排障」的内容，见同系列的《MCP 实践：配置、传输与安全清单》。
